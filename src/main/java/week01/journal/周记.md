## 困难
1. "userMapper.xml"文件路经mybatis配置文件中写错了，导致无法连接数据库，报错"Invalid bound statement (not found): com.example.mapper.UserMapper.insertUser"，后来才发现是路径写错了，改成"com/example/mapper/UserMapper.xml"就好了。
2. "UserMapper.java"接口中方法名和"UserMapper.xml"文件中SQL语句的id不一致，导致无法执行SQL语句，报错"Invalid bound statement (not found): com.example.mapper.UserMapper.insertUser"，后来才发现是方法名和id不一致，改成一致就好了。
3. Student在继承User类后，多了楼号和房号两个属性，对应的数据库处理也需要修改
4. 在service层调用mapper时总是要调用到factory.getMapper()方法，代码复用率低，于是创建了个函数来专门创建factory
5. 因为service层只处理逻辑，尽量不要出现交互，可是在一些方法的调用中，可能会有不同的错误需提示，如果只凭借返回值来判断，可能会比较麻烦
    <br>**解决方法:** throw 异常，在service层抛出异常，在调用service层的函数时捕获异常并进行处理  
6. 用基础数据在User表 细节数据分别放在Admin和Student表，在查询时需要进行多表查询，比较麻烦
    <br>**解决方法:** 在mapper中使用join语句进行多表查询以Id来连接表
